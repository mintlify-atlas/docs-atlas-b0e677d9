AI DevKit is designed to work across the most widely used AI coding agents. Each supported agent gets its own configuration files, context documents, and command templates — all generated automatically by `ai-devkit init`. The table below shows which features are available for each agent.

## What each feature means

**Agent Setup** means AI DevKit can write configuration files, context documents, and command templates for that agent when you run `ai-devkit init`. Agents marked as "Supported" are stable and actively maintained. Agents marked as "Testing" are functional but may have rough edges — feedback is welcome.

**Agent Control** means AI DevKit can interact with a running agent session from the CLI — for example, sending instructions, inspecting the active conversation, or monitoring agent progress. This feature requires the agent to expose a supported control interface. Currently only Claude Code and Codex CLI meet this requirement.

## Selecting agents during init

When you run `npx ai-devkit@latest init`, you choose one or more agents interactively:

```bash
npx ai-devkit@latest init
```

You can also specify agents directly using the `--environment` flag with a comma-separated list of environment codes:

```bash
npx ai-devkit@latest init --environment claude,cursor,codex
```

Valid environment codes are: `claude`, `cursor`, `github`, `gemini`, `codex`, `opencode`, `antigravity`, `windsurf`, `kilocode`, `roo`, `amp`.
