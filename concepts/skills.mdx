A skill is a directory containing a `SKILL.md` file. That file holds specialized instructions, workflows, or reference material that an AI agent reads to gain domain-specific knowledge — for example, a TDD workflow, a debugging methodology, or a technical writing guide. Skills let you encode best practices once and share them across every agent and every project.

## How skills work

Skills live in GitHub repositories called registries. AI DevKit indexes these registries and installs individual skills by symlinking (or copying) the skill directory into each agent's designated skill folder.

<Steps>
  <Step title="Find a skill">
    Search across all registered skill sources by keyword. The index is fetched automatically on first use and refreshed every 24 hours.
    ```bash
    ai-devkit skill find tdd
    ```
  </Step>
  <Step title="Install a skill">
    Install by specifying the registry identifier and skill name. AI DevKit clones the registry, locates the skill directory, and symlinks it into each configured agent's skill folder.
    ```bash
    ai-devkit skill add codeaholicguy/ai-devkit dev-lifecycle
    ```
  </Step>
  <Step title="Agent reads the skill">
    Once installed, the agent finds the `SKILL.md` in its skill directory and incorporates the instructions into its working context. No further configuration is required.
  </Step>
</Steps>

## Where skills are installed

AI DevKit symlinks each skill into the skill directories of every environment configured in your project. The target path depends on the agent:

| Agent | Skill path |
|---|---|
| Claude Code (`claude`) | `.claude/skills/<skill-name>/` |
| Cursor (`cursor`) | `.cursor/skills/<skill-name>/` |
| OpenAI Codex (`codex`) | `.agents/skills/<skill-name>/` |
| AMP (`amp`) | `.agents/skills/<skill-name>/` |
| OpenCode (`opencode`) | `.opencode/skills/<skill-name>/` |
| Antigravity (`antigravity`) | `.agent/skills/<skill-name>/` |

Environments without a skill path listed (GitHub Copilot, Gemini, Windsurf, KiloCode, Roo Code) do not support local skill installation.

<Tip>
  Install a skill globally (into your home directory paths) with the `--global` flag so it applies to every project without being committed to version control.
  ```bash
  ai-devkit skill add codeaholicguy/ai-devkit tdd --global --env claude
  ```
</Tip>

## Managing installed skills

```bash
# List all skills installed in the current project
ai-devkit skill list

# Remove a skill from the current project
ai-devkit skill remove tdd

# Update all skill registries to the latest version
ai-devkit skill update

# Update a specific registry
ai-devkit skill update codeaholicguy/ai-devkit
```

## Adding custom registries

Any GitHub repository that follows the `skills/<name>/SKILL.md` directory convention can be used as a registry. Add it to your project or global configuration, then reference it by its `owner/repo` identifier:

```bash
ai-devkit skill add your-org/your-skills-repo your-skill-name
```

<Note>
  Registry metadata is cached in `~/.ai-devkit/skills.json`. If you add a new registry and do not see its skills, run `ai-devkit skill find <keyword> --refresh` to rebuild the index.
</Note>
