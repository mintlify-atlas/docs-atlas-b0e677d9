---
title: "Quickstart: configure AI DevKit for your project"
description: "Install AI DevKit and configure your first AI coding agent in under a minute. Covers init, environment selection, phase setup, and first skill install."
---

AI DevKit sets up your project for AI-assisted development through an interactive wizard. The steps below walk you from a blank directory to a fully configured workspace with agent templates, phase documents, and your first installed skill.

<Note>
  AI DevKit requires Node.js 20.20.0 or higher. Run `node --version` to check your version before proceeding.
</Note>

<Steps>
  <Step title="Run the init wizard">
    In your project root, run:

    ```bash
    npx ai-devkit@latest init
    ```

    This launches an interactive setup wizard. If AI DevKit has not been initialized in this directory before, the wizard proceeds immediately. If a configuration already exists, it asks whether you want to reconfigure.
  </Step>

  <Step title="Select AI environments">
    The wizard prompts you to select one or more AI coding agents to configure. Use the spacebar to select, then Enter to confirm.

    ```
    ? Select AI environments to set up (use space to select, enter to confirm):
     ◯ Claude Code
     ◯ Cursor
     ◯ GitHub Copilot
     ◯ Gemini CLI
     ◯ Codex CLI
     ◯ opencode
     ◯ Antigravity
     ◯ Windsurf
     ◯ Kilo Code
     ◯ Roo Code
     ◯ Amp
    ```

    Each selected environment gets a dedicated context file and command configuration written to your project. You can add more environments later by running `npx ai-devkit init` again.
  </Step>

  <Step title="Select development phases">
    Next, choose which development phases to enable. Each phase creates a structured context document in `docs/ai/` that guides the agent through that stage of work.

    ```
    ? Select phases to initialize:
     ◯ Requirements & Problem Understanding
     ◯ System Design & Architecture
     ◯ Project Planning & Task Breakdown
     ◯ Implementation Guide
     ◯ Testing Strategy
     ◯ Deployment Strategy
     ◯ Monitoring & Observability
    ```

    To enable all phases at once, use the `--all` flag:

    ```bash
    npx ai-devkit@latest init --all
    ```

    You can add individual phases later with:

    ```bash
    npx ai-devkit phase <name>
    ```

    Where `<name>` is one of: `requirements`, `design`, `planning`, `implementation`, `testing`, `deployment`, `monitoring`.
  </Step>

  <Step title="Add a skill from the registry">
    Skills extend agents with reusable workflows and specialized knowledge. After initialization completes, install your first skill:

    ```bash
    npx ai-devkit skill add codeaholicguy/ai-devkit dev-lifecycle
    ```

    Skills are written as SKILL.md files and placed in the appropriate directory for each configured agent. To see available skills in a registry, browse the registry directly or use the skill commands in the [CLI reference](/cli/skill).
  </Step>

  <Step title="Verify with lint">
    Run the lint command to confirm your workspace is correctly configured:

    ```bash
    npx ai-devkit lint
    ```

    The lint command checks that all expected configuration files are present and that your workspace meets AI DevKit's requirements. A clean output means your agents are ready to use.

    Example output:

    ```
    ✔ Configuration file found
    ✔ Environment templates present
    ✔ Phase documents present
    ✔ Workspace ready for AI DevKit workflows
    ```
  </Step>
</Steps>

## What gets created

After initialization, your project will contain:

- **`.ai-devkit.json`** — The AI DevKit configuration file tracking your selected environments, phases, and installed skills.
- **`docs/ai/`** — Phase context documents (e.g., `requirements.md`, `implementation.md`) that agents reference during development.
- **Agent configuration files** — Environment-specific context and command files for each agent you selected (e.g., `CLAUDE.md` for Claude Code, `.cursor/rules/` for Cursor).
